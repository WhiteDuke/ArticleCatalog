### Web-приложение, предоставляющее HTTP API для каталога статей.

Для корректной работы приложения необходим запущенный экземпляр СУБД PostgreSQL.
Экземпляр может быть развёрнут на локальной машине или в контейнере Docker.
Для развёртывания в Docker в проект добавлен файл **ArticleCatalog.Service/compose.yaml**.

В проекте есть файлы миграции, в которой создаются база данных ArticleCatalog и все необходимые таблицы.
Миграции доступны в каталоге **ArticleCatalog.DataAccess/Migrations**.
Миграции будут автоматически запущены при старте приложения.

Web приложение может быть запущено в нескольких конфигурациях.
Одно из них - http.
В этом случае swagger приложения доступен по адресу http://localhost:5041/swagger/index.html.
На этой странице доступны запросы получения списка статей, создания статьи, редактирования статьи, 
получения списка разделов, получения списка статей по конкретному разделу.

Для запуска приложения в контейнере Docker необходимо собрать образ.
В терминале нужно перейти в корень решения и выполнить команду (без апострофов):  
**'docker build -t article-catalog:latest .'**

Для запуска приложения в контейнере необходимо выполнить команду (без апострофов):  
**'docker compose up -d'**.  
Будет развернут сервер PostgreSQL, запущен контейнер с приложением.  
Обратите внимание: в файле **ArticleCatalog\compose.yaml** закомментированы строчки:  
\#volumes:  
\#- "./postgredb-data:/db-data"  
они нужны для монтирования каталогов между хост-системой и контейнером.
Если раскомментировать эти строки, то данные СУБД PostgreSQL будут сохраняться в файловой системе хоста в каталоге postgredb-data.
При этом каталог postgredb-data будет создан в том же каталоге, в котором находится файл compose.yaml.
Таким образом данные СУБД будут сохранены и не будут удалены при перезапуске контейнера.

Swagger приложения также будет доступен в браузере по адресу:  
**http://localhost:5041/swagger/index.html**
